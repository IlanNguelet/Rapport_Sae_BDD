Introduction

Ce rapport présente une analyse complète des données de KDou, une entreprise ayant pour objectif d'optimiser son offre de produits et d’améliorer ses relations avec ses clients et ses fournisseurs. Ce travail vise à fournir des recommandations concrètes basées sur des indicateurs pertinents extraits de la base de données de l'entreprise, en utilisant des outils tels que le SQL (Structured Query Language) et le QBE (Query By Example) sous Access pour une analyse approfondie.

Objectifs et Structure du Rapport

Le rapport est divisé en plusieurs sections correspondant aux différentes missions d'analyse. Chaque mission aborde un aspect spécifique des données de KDou et propose des indicateurs clés pour guider la prise de décision. Les pages principales sont les suivantes :

  • Mission 1 : Analyse préliminaire des données pour une compréhension générale des informations disponibles et des premières observations.
  • Mission 2 : Exploitation de la base de données via une interface low-code (Access), permettant de créer des indicateurs clés sans nécessiter de programmation complexe.
  • Mission 3 : Utilisation de requêtes SQL avancées pour une analyse ciblée des catégories de produits, de l’offre de KDou, de la provenance des produits, et de la clientèle.
  • Conclusion : Synthèse des recommandations pour KDou, issues des analyses effectuées, pour aider l'entreprise à ajuster sa stratégie.
  • Rédaction du Rapport : Explication des choix techniques et de la structure du site.

Outils et Méthodes Utilisés

Pour mener à bien cette analyse, plusieurs outils et méthodologies ont été utilisés :

  • SQL : Un langage de requête structuré pour extraire, filtrer et analyser les données de manière précise. Les requêtes SQL nous ont permis de créer des indicateurs pertinents, par exemple, pour identifier les produits non commandés ou les produits à forte demande.
  • Microsoft Access : Une interface low-code pour la gestion et l'interrogation de la base de données. Access a été particulièrement utile pour les utilisateurs ayant peu d'expérience en programmation, permettant de construire des requêtes de manière visuelle.
  • HTML et CSS : La structure du rapport a été mise en place avec HTML pour organiser le contenu, et CSS pour une présentation soignée et cohérente, adaptée à différents types d’écrans grâce à des media queries.

Ces outils ont permis une analyse des données flexible et accessible, tout en garantissant la précision et la cohérence des résultats.
