Synthèse Générale

Ce rapport a été réalisé dans le cadre de l’analyse des données de KDou. La mission comprend trois analyses distinctes sur les données, chacune ayant ses propres objectifs et indicateurs. Le site HTML a été structuré pour faciliter la navigation et la présentation de chaque mission de manière claire, permettant aux utilisateurs de comprendre et d’explorer chaque partie de l’analyse.

Structure des Pages

Chaque mission possède sa propre page HTML, organisée en sections distinctes avec des titres pour chaque indicateur. Cette structuration permet une lecture intuitive et un accès rapide aux résultats et interprétations :

  • Accueil (index.html) : Présente le contexte et les objectifs globaux de l’analyse.
  • Mission 1, Mission 2, Mission 3 : Chaque page (mission1.html, mission2.html, mission3.html) détaille les objectifs, les indicateurs, les résultats, et les interprétations spécifiques à la mission.
  • Conclusion : Résume les axes d’amélioration et les recommandations pour KDou suite à l’analyse.
  • Rédaction du Rapport : Fournit une explication des choix techniques et de la structure du site HTML et CSS.

Choix Techniques et Explications

Organisation des Fichiers CSS

Chaque page possède un fichier CSS distinct afin de permettre un contrôle précis sur le style de chaque page. Les styles de base (couleurs, polices, espacements) sont unifiés pour garantir une cohérence visuelle sur l'ensemble du site.

  • Couleurs de chaque page : Chaque fichier CSS utilise une couleur dominante unique pour les en-têtes et la navigation, facilitant l’identification de la page par l’utilisateur.
  • Navigation : Le menu de navigation utilise des liens qui affichent un soulignement au survol, offrant une meilleure interaction utilisateur.
  • Media Queries : Des media queries sont intégrées dans chaque fichier CSS pour ajuster la mise en page en fonction de la taille de l'écran, garantissant ainsi une lisibilité optimale sur ordinateurs, tablettes et smartphones.

Structure HTML et Accessibilité

Les pages HTML sont organisées avec des balises sémantiques pour faciliter l’accessibilité et la lisibilité :

  • <header> : Contient le titre de chaque page avec un en-tête distinctif pour identifier la section en cours.
  • <nav> : La barre de navigation utilise des balises <a> et des liens pour pouvoir changer de page.
  • <main> : Contient le contenu principal, structuré en sections avec des sous-titres pour chaque indicateur.
  • <section> : Chaque indicateur est placé dans une balise <section> pour une hiérarchisation logique et une meilleure accessibilité.
  • <footer> : Utilisé pour le copyright et les informations de bas de page, uniforme sur toutes les pages.

Présentation des Indicateurs et des Résultats

Les résultats sont présentés sous forme de tableaux d’images et de blocs de code pour les requêtes SQL (captures d'écran à partir de la base de données sous Access). Cette approche permet d’améliorer la lisibilité et de fournir un format visuel adapté aux utilisateurs non techniques. Les balises <img> sont utilisées pour afficher les résultats et le code SQL sous forme d’images, permettant une visualisation rapide des données.

Explication des Styles CSS Clés

Barre de Navigation

La barre de navigation utilise un style sobre avec des effets de survol et un état actif. Les liens sont centrés et espacé pour améliorer la lisibilité.


/* Style de base pour la barre de navigation */
nav ul {
    display: flex;
    justify-content: center;
    list-style-type: none;
    background-color: #333;
    padding: 10px;
    margin: 0;
}

nav ul li {
    margin: 0 15px;
}

nav ul li a {
    color: white;
    text-decoration: none;
    font-weight: bold;
    padding: 5px;
    transition: color 0.3s ease;
}

/* Effet au survol pour tous les liens */
nav ul li a:hover {
    color: #ffde59;
}

Ce style ajoute une couleur distincte et un soulignement au lien survolé. L'effet de survol (:hover) change la couleur pour indiquer au visiteur que le lien est cliquable.

Media Queries pour Responsivité

Les media queries permettent de modifier la mise en page en fonction de la taille de l’écran, pour une expérience fluide sur tous les appareils.


/* Media Query pour les smartphones */
@media screen and (max-width: 767px) {
    nav ul {
        flex-direction: column;
        padding: 0;
    }

    nav ul li {
        margin: 10px 0;
    }

    nav ul li a {
        font-size: 14px;
    }

    main {
        padding: 10px;
    }

    header, footer {
        padding: 10px;
        text-align: center;
    }
}

Ce code utilise des media queries pour ajuster la mise en page. Sur les petits écrans, le menu devient vertical et la taille des polices est réduite pour maximiser l’espace.

Pour les autres styles CSS, nous avons regardé dans le cours ou sur Internet la documentation des balises correspondantes à l'effet que nous voulions créer.
